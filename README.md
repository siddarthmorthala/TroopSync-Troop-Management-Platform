# TroopSync

TroopSync is a modern full-stack troop operations platform commissioned by the leadership of **Troop 261** in **Plano, TX** to replace fragmented spreadsheets, text chains, and manual record keeping with a single system for advancement, planning, communication, finances, and service tracking.

**Developer:** Siddarth Morthala

## Why this project exists
Troop leadership needed a better way to manage advancement, roster records, campouts, fundraisers, scout accounts, and communication without bouncing between spreadsheets, Scoutbook exports, and manual reminders. TroopSync centralizes those workflows in one web app designed for leaders, parents, and scouts.

## Core features
- **Advancement tracking** for ranks and merit badges with built-in BSA-style rules validation
- **Roster management** for scouts and adults with role-based access control
- **Activity, fundraiser, and attendance tracking**
- **Scout account ledger** for troop treasurers
- **Import support** for Scoutbook and Internet Advancement style CSVs
- **Collaborative event planning** with notes, tasks, and assignments
- **Campout planning** with gear checklists, duty rosters, and trip logistics
- **Parent communication** with event reminders and message templates
- **Carpool coordination** with seat counts and driver assignments
- **Meal planner** with autogenerated shopping lists
- **Service hour logger** with gamified leaderboards
- **Interactive merit badge guide** with plain-language requirement summaries and resource links
- **Internal reporting dashboard** for advancement, service, attendance, and financial snapshots

## Tech stack
- **Frontend:** React, TypeScript, Vite
- **Backend:** FastAPI, SQLAlchemy, Pydantic
- **Database:** PostgreSQL
- **Infra:** Docker, Docker Compose, AWS deployment notes
- **Testing:** pytest, httpx
- **CI/CD:** GitHub Actions

## AWS deployment notes
The app is designed to be containerized and deployed using:
- AWS EC2 for web containers
- AWS RDS for PostgreSQL
- AWS ALB or Nginx reverse proxy for routing
- GitHub Actions for CI on push

## Testing
The backend includes integration-style tests for:
- advancement rule checks
- double counting prevention in attendance/service logging
- ledger transaction balancing
